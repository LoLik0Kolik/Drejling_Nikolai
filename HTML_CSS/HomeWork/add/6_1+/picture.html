При создании адаптивного дизайна, могут возникнуть проблемы с картинками. Если нам надо использовать картинку, то у нее уже есть собственные "height" и "width", допустим 700 на 700 px. При изменении разрешения экрана, ширина будет либо расширяться, либо сжиматься. Это проблема, ибо у большинства изображений уже есть базовая ширина, которую нельзя никак изменять. Решение предлагает тег
"picture"
.
"Picture"
это новый элемент, который является частью HTML5. С помощью
"Picture"
можно заменять изображения на другие, более подходящие для конкретного устройства (в зависимости от результатов media-выражений и плотности пикселей). Основные шаги при работе с
"Picture"
:
  1. Создайте открывающий и закрывающий теги
    "Picture"
  2. Внутри создайте
    "Source"
    элемент для каждого выражения, что хотите обработать.
  3. Добавьте атрибут
    media
    , содержащий выражения для таких вещей как высота и ширина области просмотра, ориентация и т.д.
  4. Добавьте атрибут
    srcset
    с соответствующим именем файла изображения для загрузки.
  5. Добавьте дополнительные имена файлов к атрибуту srcset, если нужно поддерживать разную плотность пикселей, например для Retina дисплеев.
  6. Добавьте резервный (fallback) элемент .
  7. Атрибут
    type
    позволяет вам обозначить MIME-тип данных, указанных в атрибуте srcset элемента
    "source"
    . Если браузер не поддерживает данный тип, то элемент "source" опускается.
